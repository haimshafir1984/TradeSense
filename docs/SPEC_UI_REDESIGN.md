# מסמך איפיון: עיצוב מחדש של הממשק — "מסך עבודה" במקום "דף נחיתה"

תאריך: 2026-07-09
סטטוס: מאושר לביצוע
מבצע מיועד: סוכן קוד אוטונומי (Sonnet)
תחולה: **client בלבד** — אסור לגעת בשום קובץ תחת `server/`.

---

## 0. הרציונל

המשתמש עובד עם המערכת בעיקר ב-web במחשב, נכנס אליה כל בוקר כדי לראות את
"רשימת המעקב למחר" ולבצע סריקות. העיצוב הנוכחי בנוי כדף נחיתה שיווקי: hero
ענק (h1 עד 4.4rem), רקע עם שני כתמי aurora מרחפים ו-radial-gradients, כרטיסים
עם `backdrop-filter: blur` ורדיוסים של 28px, והכול בעמוד גלילה אחד ארוך
(סריקה → תוצאות → watchlist → תיק). למשתמש יומיומי זה רעש ויזואלי וגלילה
מיותרת.

היעד: מסך עבודה שקט, דחוס במידה, מהיר להתמצאות — בלי לאבד שום פונקציונליות.

## 1. עקרונות מחייבים

1. **אפס שינוי בהתנהגות עסקית.** כל ה-state, ה-fetch, וה-handlers הקיימים ב-
   `App.jsx` ו-`PortfolioSection.jsx` נשארים כמו שהם. משנים מבנה תצוגה
   ועיצוב בלבד. אסור לשנות שום route או פורמט request/response.
2. **RTL ועברית נשמרים בכל מקום.** כל הטקסטים הקיימים נשארים; מותר לקצר
   טקסטים תיאוריים אם הפריסה החדשה דורשת, אבל לא למחוק מידע (למשל
   הדיסקליימר של ה-watchlist חייב להישאר בולט).
3. **בלי ספריות חדשות.** לא UI kit, לא ספריית טאבים, לא framework ל-CSS.
   React + CSS קיים בלבד.
4. אחרי כל שלב: `npx vite build` מתוך `client/` חייב לעבור נקי (ולמחוק
   `dist` אחרי), ו-`npm test` מתוך `server/` חייב להישאר 62/62 (לא אמור
   להיות מושפע בכלל — זו בדיקת בטיחות).
5. commits עם הודעות באנגלית שמסבירות למה; בסוף — `git push origin main`.

## 2. מבנה חדש: שלושה טאבים + פס עליון

### 2.1 פס עליון (top bar)

פס דק וקבוע (sticky top) שמכיל, מימין לשמאל (RTL):
- שם המערכת "TradeSense" (טקסט, לא לוגו; גודל ~1.1rem, משקל 800)
- שלושת הטאבים (ראו 2.2)
- בצד השמאלי: ה-badges הקיימים כשהם זמינים — מצב שוק (regime), ו"מובילה
  ב-90 הימים" (ליגת האסטרטגיות). אלה כבר קיימים היום כ-state ב-App.jsx —
  רק להעביר אותם לפס העליון.

ה-hero הנוכחי (eyebrow + h1 ענק + פסקת שיווק) — **נמחק**. שם המערכת בפס
העליון מחליף אותו.

### 2.2 שלושת הטאבים

ניווט client-side פשוט עם `useState` (בלי router):

1. **"רשימת מעקב למחר"** — **טאב ברירת המחדל בכניסה.** מציג את פאנל ה-
   watchlist הקיים: חותמת "עודכן לאחרונה", chip מקור הנתונים, כפתור "רענן
   עכשיו", הדיסקליימר, והטבלה. הטעינה האוטומטית הקיימת (useEffect על
   form.exchange) נשארת.
2. **"סריקת שוק"** — הטופס + התוצאות (ראו 2.3).
3. **"התיק שלי"** — `PortfolioSection` הקיים כמו שהוא.

מעבר טאב לא מאבד state (כל הטאבים mounted, מוסתרים ב-CSS או ברינדור
מותנה — אם רינדור מותנה, לוודא שה-state המרכזי נשאר ב-App ולא נמחק
במעבר; ה-state כבר שם היום, אז רינדור מותנה בסדר).

בורר הבורסה (exchange) משפיע גם על ה-watchlist וגם על הסריקה — להעביר
אותו לפס העליון (select קומפקטי) כך שיהיה זמין מכל טאב, ולהסיר אותו
מהטופס. שאר שדות הטופס נשארים בטאב הסריקה.

### 2.3 פריסת טאב "סריקת שוק" במסך רחב

Grid דו-עמודתי מ-`min-width: 960px`:
- **עמודה צרה (imin ~300px, בצד ימין ב-RTL):** הגדרות הסריקה — רמת סיכון,
  שיטת השקעה (עם התיאורים הקיימים), כפתור "סרוק שוק". **הפילטרים
  המתקדמים עוברים ל-`<details>` מקופל כברירת מחדל** עם summary "פילטרים
  מתקדמים". העמודה sticky (top מתחת לפס העליון) כדי שהכפתור תמיד נגיש.
- **עמודה רחבה:** meta-bar (מקור נתונים, מצב שוק, הערות ההמלצה) + טבלת
  התוצאות.

מתחת ל-960px: עמודה אחת, הטופס מעל התוצאות (כמו היום).

### 2.4 טבלאות

בשתי הטבלאות (תוצאות + watchlist):
- `font-variant-numeric: tabular-nums` על תאי מספרים
- `thead` דביק (`position: sticky; top: 0`) בתוך wrapper הגלילה הקיים
- ריווח שורות צפוף יותר: padding אנכי ~0.6rem במקום 1rem
- שינוי יומי חיובי בירוק הקיים, שלילי באדום הקיים (class קיים value-tone)
- שורות זברה עדינות (רקע לסירוגין בשקיפות נמוכה) לקריאה רוחבית

## 3. שפה ויזואלית חדשה (styles.css)

### 3.1 מה נמחק

- שני אלמנטי ה-`.aurora` (גם מה-JSX) וה-radial-gradients מרקע ה-body
- כל `backdrop-filter: blur`
- ה-hero וכל חוקי ה-CSS שלו
- הצל הכבד `--shadow: 0 24px 80px ...`

### 3.2 טוקנים חדשים

```css
:root {
  --bg: #f6f7f9;            /* רקע אחיד שקט, אפור-כחלחל בהיר */
  --panel: #ffffff;
  --text: #1a2333;
  --muted: #5e6b81;
  --accent: #0b8f6a;        /* הירוק הקיים נשאר — צבע ההדגשה היחיד */
  --accent-strong: #0a6d59;
  --negative: #b83535;      /* לאדום של ירידות/שגיאות */
  --border: #e2e6ec;
  --shadow: 0 1px 2px rgba(26, 35, 51, 0.06);
  --radius: 12px;           /* במקום 28px */
}
```

- כרטיסים: רקע `--panel`, מסגרת `1px solid var(--border)`, רדיוס
  `--radius`, הצל העדין החדש.
- כפתור ראשי ("סרוק שוק", "רענן עכשיו"): מלבני פשוט ברדיוס 8px, רקע
  `--accent`, בלי גרדיאנט ובלי אנימציית ריחוף מוגזמת (מותר שינוי גוון עדין).
- ה-pills/badges הקיימים (metric-pill, source-badge וכו') נשארים בקונספט,
  רק מותאמים לטוקנים החדשים.
- טיפוגרפיה: להישאר עם ערכת הגופנים הקיימת. כותרות סקשן ~1.15rem משקל
  700; אין יותר כותרות ענק.

## 4. סדר ביצוע מומלץ (commits)

1. **טאבים ופס עליון:** הוספת ה-top bar, שלושת הטאבים, העברת exchange
   selector וה-badges לפס העליון, מחיקת ה-hero וה-aurora מה-JSX.
2. **פריסת טאב הסריקה:** grid דו-עמודתי, details לפילטרים, sticky

3. **שפה ויזואלית:** החלפת הטוקנים, ניקוי ה-CSS המת (aurora/hero/blur),
   עדכון טבלאות (sticky header, zebra, tabular-nums, צפיפות).
4. עדכון README קצר (סעיף Frontend: תיאור שלושת הטאבים) — commit נפרד.

## 5. קריטריוני קבלה

1. `npx vite build` נקי; 62/62 טסטים בשרת ללא שינוי.
2. שלושת הטאבים עובדים; ברירת המחדל היא "רשימת מעקב למחר"; מעבר טאבים
   לא מאבד תוצאות סריקה שכבר התקבלו.
3. בורר הבורסה בפס העליון משפיע גם על הסריקה וגם על ה-watchlist.
4. אין שום קריאת רשת חדשה ואין שינוי בשום payload.
5. אין `aurora`, `backdrop-filter`, או `hero` ב-codebase של ה-client.
6. הפריסה תקינה גם ב-1280px, גם ב-960px, וגם במובייל 375px (עמודה אחת).
7. הדיסקליימר של ה-watchlist נשאר גלוי ובולט בטאב שלו.
